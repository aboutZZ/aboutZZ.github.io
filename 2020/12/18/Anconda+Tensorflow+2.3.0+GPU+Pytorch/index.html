在本文中，您将看到如何在Anconda下如何进行Tensorflow GPU版本的安装配置。

Anconda 安装 Tensorflow 2.3.0 GPU + Pytorch

安装CUDA

  • 首先你需要去确认下你的GPU是否支持CUDA（A卡抱歉）

  • 去微软下载VC++运行时，如果已安装请跳过。

  • 去Nvidia官网下载CUDA安装包，Tensorflow 2.3.0 最高支持 10.1 版本，因此我们找到 CUDA Toolkit 10.1 update2 (Aug 2019) 进行下载。注意，由于CUDA里面集成了显卡驱动，且是老版本了，在安装时我们可以选择自定义安装，取消勾选显卡驱动项。

  • 去Nvidia官网下载cuDNN，这里需要登录才可以下载，没有账号的注册个账号即可。下载列表中选择Download cuDNN v7.6.5 (November 5th, 2019), for CUDA 10.1，注意不要选错版本了，要对应 CUDA 10.1。下载下来是一个压缩包，解压后有个cuda文件夹，我们把该文件夹里面的所有内容转移至CUDA安装文件夹即可。【注】CUDA默认安装位置为 C:\Program Files\NVIDIA GPU Computing Toolkit\CUDA\v10.1

安装Anconda

去 Anconda官网下载并安装，安装过程中有以下几点需要注意：

  • 安装路径中不能含有空格。
  • 尽量选择只为我安装（这样不需要管理员权限，不会弹出UAC），否则后面安装依赖会遇到权限问题。
  • Advanced Installation Options中不要勾选Add Anaconda to my PATH environment variable.可能会影响其他程序及Anconda的卸载。

Anconda虚拟环境配置

这里先说明下为什么需要创建独立的环境。比如你在A项目中用了 Python 2，而新的项目B使用Python 3，而同时安装两个Python版本可能会造成许多混乱和错误。这时候 conda就可以帮助你为不同的项目建立不同的运行环境，环境之间是相互隔离的，互不影响。

还有很多项目使用的包版本不同，比如不同的pandas版本，不可能同时安装两个 Numpy 版本，你要做的应该是，为每个 Numpy 版本创建一个环境，然后项目的对应环境中工作。这时候conda就可以帮你做到。

环境创建

打开开始菜单，找到或直接搜索Anaconda Promot，打开Anaconda命令行，输入以下命令来创建一个环境：

# TensorFlow 2.3.0 需求 Python 版本为 3.5 - 3.7，因此这里需要指定 python 版本为 3.7，否则默认是 3.8
$ conda create -n tensorflow python=3.7

接着，激活该环境：

$ conda activate tensorflow

依赖安装

# 首先升级一下 pip，因为 TensorFlow 2 软件包需要使用高于 19.0 的 pip 版本。
$ pip install --upgrade pip
# 安装TensorFlow GPU版本
# 下载速度在国内很慢，我实际操作时使用了代理：pip install tensorflow-gpu==2.3.0 --proxy=127.0.0.1:1081
$ pip install --upgrade tensorflow-gpu==2.3.0
# 安装Torch、CUDA
$ conda install pytorch torchvision cudatoolkit=10.1 -c pytorch

恭喜，到这里你已经成功安装！

校验安装是否成功

接下来我们测试下GPU是否开启，输入python进入python：

$ python

接下来输入下面的代码测试GPU是否工作：

import tensorflow as tf
tf.config.list_physical_devices('GPU')
