使用Github Pages + Hexo创建自己的博客小站

在本教程中，我们将一步步教您将 Hexo 博客部署到 GitHub Pages 上

准备工作

安装Git

去git官网下载安装git，如果你不清楚安装过程中各选项意义，一路下一步即可。

安装Node.js及Hexo

  • 去Node.js官网下载并安装，建议使用LTS（长期支持）版本。

  • 安装Hexo

    打开Windows命令提示符，键入以下命令，安装hexo脚手架

$ npm install hexo-cli -g
  • 创建Hexo工程

在命令提示符中，继续键入以下命令

$ hexo init

这样，一个初级的博客就建立好了，这时我们继续键入命令来看看效果

$ hexo g
INFO  Validating config
INFO  Start processing
INFO  Files loaded in 168 ms
INFO  0 files generated in 69 ms

$ hexo s
INFO  Validating config
INFO  Start processing
INFO  Hexo is running at http://localhost:4000 . Press Ctrl+C to stop.

打开浏览器，打开网址：http://localhost:4000，你就可以看到博客页面了。

部署至Github Pages

配置SSH key

如果您已经生成过ssh秘钥，此步骤可跳过。

  • 打开Windows命令提示符，键入以下命令来生成ssh密钥对
$ ssh-keygen

首先，会询问你保存的位置，这里我们直接按回车，选择默认位置，保存在当前用户文件夹下。

Generating public/private rsa key pair.
# Enter file in which to save the key (/c/Users/you/.ssh/id_rsa): [Press enter]

其次，询问你密码，如果不需要密码，直接按下回车，否则输入密码后按回车

Enter passphrase (empty for no passphrase): 
# Enter same passphrase again:

  • 接着，打开Windows资源管理器，在地址栏输入%USERNAME%\.ssh并回车，用记事本等工具打开其中的id_rsa.pub文件

  • 登录Github，在右上角点击你的头像，点击Settings，打开设置页面

在左侧菜单点击SSH and GPG keys，点击New SSH key，在Key这一栏中复制粘贴第2步获得的id_rsa.pub文件内容，title栏可以随意起一个名字。

至此，SSH秘钥部署完成。

创建您的Repository

在您的Github主页，点击右上角加号，并点击New repository按钮，在Repository name栏中，填入<你的 GitHub 用户名>.github.io（以下请将<你的 GitHub 用户名>替换成您自己的用户名）。这样建立后，在浏览器就可以通过<你的 GitHub 用户名>.github.io直接访问您的博客。

部署到Github Pages

打开博客根目录下的_config.yml文件，在最后找到deploy，设置您的repo路径：

deploy:
  type: git
  repository: https://github.com/<你的 GitHub 用户名>/<你的 GitHub 用户名>.github.io
  branch: master

打开Windows 命令提示符，并cd至您的博客根目录，执行命令：

# 安装部署工具
$ npm install hexo-deployer-git --save
# 清理缓存
$ hexo clean
# 重新生成网页静态文件
$ hexo g
# 部署
$ hexo d

部署成功后，等待一分钟，使用浏览器访问<你的 GitHub 用户名>.github.io
